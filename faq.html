FAQs

Common questions about HTML Editor.

What browsers are supported?

Chrome and Edge support all features including direct file saving via the File System Access API. Firefox and Safari can open and edit files but will download a copy on save instead of writing back to the original file.

Does my file get uploaded anywhere?

No. Everything happens entirely in your browser. Your file is never sent to any server — not even ours. HTML Editor has no backend.

What can I edit?

You can edit any visible text directly on the page. HTML Editor uses contentEditable — the same mechanism browsers use for rich text fields — so you click anywhere and type.

Can I edit images, styles, or code?

Not with inline editing. HTML Editor is designed for quick text tweaks, not code editing. For structural or style changes, use your code editor and reload the file.

How does Save work?

On Chrome/Edge, Save writes directly back to the file you opened — no download, no rename. Save As lets you choose a new location. On Firefox/Safari, both options download a copy.

What are the keyboard shortcuts?

⌘O Open  ·  ⌘E Toggle edit mode  ·  ⌘S Save
